# JADE规范 jade作为一个高性能的前端模板引擎，使用javascript实现，可以在node环境使用。提供一个试玩jade的网站，可以里面体验一下jade的语法的便利之处：[jade试玩](http://naltatis.github.io/jade-syntax-docs/) 以下是jade的一些使用规范 ### 一.标签 jade放弃了传统的尖括号标签时代，使用单词的形式就代表一个标签 ####文档声明 **作为文档的根元素，来告知浏览器当前的页面用的哪一种html** 例如： html5的页面可以直接使用 ``` doctype html ``` 效果如下： ``` <!DOCTYPE html> ``` #### 标签及文本 **标签可以直接使用单词来表示；页面中的标签如果是并列关系，直接换行书写；如果是包含关系，则需要在下面一行缩进，再书写；标签与里面的文本中间需要空一格** 例如： ``` doctype html html body div h1 我是标题 p 我是p标签 span 我是p标签里面的span ``` 生成的html文件是这样的： ``` <!DOCTYPE html> <html> <body> <div> <h1>我是标题</h1> <p> 我是p标签<span> 我是p标签里面的span</span></p> </div> </body> </html> ``` 对于大段的文本，可以这么写： ``` p | foo bar | hello world ``` 效果如下： ``` <p> foo bar hello world </p> ``` *** ### 二.元素的属性 **元素属性是需要写到标签后面，并且用括号包起来** 但是当一个值是 undefined 或者 null 属性不会被加上,所以它不会编译出 something="null" ``` a(href='javascript:;' class='btn') ``` 渲染为： ``` <a href="javascript:;" class="btn"></a> ``` **对于我们常用的class和id，jade给我们提供了一个便利的写法，可以直接写到标签后面，如果有多个class的话，直接写就可以，不需要加空格** **标签和属性之间没有空格** 例如： ``` div.content.box h4.title.title1#title2 我是h4 ``` 效果如下： ``` <div class="content box"> <h4 id="title2" class="title title1">我是h4</h4> </div> ``` **注：如果没有写标签，直接写id或者class，jade会默认添加一个div** 例如： ``` .content#box ``` 效果如下： ``` <div id="box" class="content"></div> ``` *** ### 三.注释 **单行注释和 JavaScript 里是一样的，通过 // 来开始，并且必须单独一行：** ``` // hello world p foo p bar ``` 渲染为： ``` <!-- hello world --> <p>foo</p> <p>bar</p> ``` **jade支持不输出的注释:** ``` //- 看不见我 p foo p bar ``` 渲染为： ``` <p>foo</p> <p>bar</p> ``` **块注释** ``` body // #content h1 Example ``` 渲染为： ``` <body> <!-- <div id="content"> <h1>Example</h1> </div> --> </body> ``` *** ### 四.变量 变量的写法： ``` - var foo = "hello world" h1= foo ``` 渲染为： ``` <h1>hello world</h1> ``` jade输出的变量默认是转义的,如果想得到数据本身的样子，可以使用 != 或者 ! *** ### 五.循环遍历 **jade尽管已经支持JavaScript原生代码，还支持了一些特殊的标签，它们可以让模板更加易于理解，其中之一就是each, 这种形式：** ``` - var items = ["one", "two", "three"] each item in items li= item ``` 渲染为： ``` <li>one</li> <li>two</li> <li>three</li> ``` **遍历一个对象的键值：** ``` obj = { foo: 'bar' } each val, key in obj li #{key}: #{val} ``` 渲染为： ``` <li>foo: bar</li> ``` *** ### 六.条件语句 **jade里面的条件语句是没有圆括号的** ``` {"name": "world"} if name == "world" h1 hello world else h1 hello bob ``` 渲染为： ``` <h1>hello world</h1> ``` *** ### 七.Mixin Mixins在编译的模板里会被Jade转换为普通的JS函数,可以带参数： ``` mixin book(name, price) li #{name} for #{price} 元 ul#books +book("Book1", 12) +book("Book2", 5) ``` 渲染为： ``` <ul id="books"> <li>Book1 for 12 元</li> <li>Book2 for 5 元</li> </ul> ``` *** ### 七.模板继承，包含和追加 1. Jade 支持通过 block 和 extends 关键字来实现模板继承。 2. 使用block定义一个块代码并调用，在页面开头使用extend xx.jade来继承某一jade文件中的block； 3. include 允许静态包含一段Jade, 或者别的存放在单个文件中的东西比如CSS, HTML，非常常见的例子是包含头部和页脚。 4. Jade允许前置和追加blocks。比如，假设在所有页面的头部都加上默认的脚本 例如： ``` html head block head script(src='/vendor/jquery.js') script(src='/vendor/caustic.js') body block content ``` 在默认的脚本之外添加一些其他的脚本，可以直接append上代码块： ``` extends layout.jade //上一页面 block append head script(src='/vendor/three.js') script(src='/game.js') ```
