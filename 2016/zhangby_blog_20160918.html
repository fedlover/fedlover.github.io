### 关于a标签的一些细节问题 构建页面时，a标签的使用率很高，但是我们并不需要每一个a标签都具有跳转页面的功能，那么就有必要总结一下，阻止a标签的默认行为有哪些。 #### 关于a标签的默认行为 1. a标签里面的href属性，里面如果填写网址，可是实现在本页面完成跳转，如：`<a href="www.baidu.com">跳转页面</a>`；如果想要在新的页面打开这个网址可以加一个target属性，如`<a href="www.baidu.com" target="_blank">跳转页面</a>`; 2. a标签的锚点定位：`<a href="#div1">锚点定位</a>`点击这个“锚点定位”的时候页面你可以跳到#div这个元素的位置。在项目中，我们可以在a标签url所对应的网址后面加上我们所要定位到的盒子id名，这样在加载页面的时候就可以跳转到此盒子的位置了。例`<a href="www.baidu.com/#div1">跳转页面</a>`； 3. 还有一些a标签处理事情不一定是要跳转页面的，很多时候仅仅是把它荡当成一个普通的按钮，利用其hover这个伪类来实现一些类似选项卡的效果。因为常用的标签li,div,span等等也都有hover这个伪类，可是他们不兼容啊，所以只能使用万能的a标签了。而关于hover这个伪类，起初我认为他只能操作元素本身或者子集，百度之后才发现，它是可以操作其他元素的,拿div的hover举例： ``` <html> <head> <style> #a:hover {color:#f60;} #a:hover > b{color:#ccc;} #a:hover + c{color:#000;} #a:hover + c > b{color:red;} </style> </head> <body> <div id='a'>元素1 <div id='b'>元素2</div> </div> <div id='c'>元素3 <div id='b'></div> </div> </body> </html> ``` #### 阻止a标签的默认行为 当我们不需要a标签实现跳转功能的时候，就需要a标签的默认行为阻止掉： 1. `<a href="javascript:;">a</a>` 2. `<a href="javascript:void 0">a</a>` 3. `<a href="var a=void(0)"></a>` 4. 也可以使用`var a=void 1;`或者`var a;`或者`var a=undefined`结果都是一样的，前三种比较常用 5. `<a href="#"></a>`可以实现回到顶部的功能，但是他的效果比较生硬，很多网站的回到顶部都是线性运动的，所以这个用处也很少； 6. 在js中阻止a的默认行为 ``` <html> <body> <a href="" id='link'>我是a标签</a> <script type='text/javascript' charset='utf-8'> var oLink=document.getElementById('link'); oLink.onclink=function(e){ e=e||window.event; //兼容IE e.preventDefault=function(){ e.returnValue=false; } } </script> </body> </html> ```
