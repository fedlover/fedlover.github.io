# Html规范 -------------- 1. 文档类型声明及编码: 统一为html5声明类型<!DOCTYPE html>， 编码统一为utf-8, 书写时利用IDE实现层次分明的缩进，禁止出现不符合W3C的标签声明出现在页面中; 2. 非特殊情况下样式文件必须外链至head之间，非特殊情况下JavaScript文件必须外链至页面底部; 3. 编码要按照XHTML标准，并且所有标签要严格闭合，比如br、input等，并且属性值一定要由双引号包括； 4. 命名规范必须由小写字母以、中划线、数字等构成，严禁出现p1、p2....p100这种无语义的命名形式； 5. 充分利用无兼容性问题的html自身标签, 比如span, em, strong, i, label等等，需要为html元素添加自定义属性的时候, 首先要考虑下有没有默认的已有的合适标签去设置, 比如头部header,尾部footer等等; 6. 语义化html, 如 标题根据重要性用h*(同一页面只能有一个h1), 段落标记用p, 列表用ul, 内联元素中不可嵌套块级元素; 7. 要尽可能的减少div的层层嵌套 ``` <div class="cim-demo"> <div class="cim-nav"> <div class="cim-name"></div> </div> </div> ``` 可以转化成这样： ``` <div class="cim-demo"> <p class="cim-nav"> <span class="cim-name"></span> </p> </div> ``` 8.书写链接地址时, 必须避免重定向，例如：href="http://www.cim120.com/", 即须在URL地址后面加上"/"； 9.在页面中禁止使用style属性,即style="…"; 10.能以背景形式呈现的图片, 尽量写入css样式中; 11.重要图片必须加上alt属性; 给重要的元素和截断的元素加上title; 12.给区块代码及重要功能(比如循环)加上注释, 方便后台添加功能; 13.书写代码的过程中，要考虑代码的向后扩展性。
